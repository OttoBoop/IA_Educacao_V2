# Bug Fix Logs

This folder contains detailed logs of bugs that were discovered and fixed in the project.

## Purpose

- Create institutional memory for debugging
- Document solutions that worked for future reference
- Track lessons learned
- Provide verification commands for similar issues

## File Naming

`YYYY-MM-DD_brief_description.md`

Examples:
- `2026-01-30_default_model_bug.md`
- `2026-01-30_chat_context_panel_bugfix.md`

## Required Sections

Each log should include:

1. **Problem** - What was broken
2. **Symptoms** - How the bug manifested
3. **Root Cause Analysis** - What caused it (and what DIDN'T)
4. **Solution Implemented** - Code changes with file paths
5. **Tests Added** - List of test functions
6. **Verification** - Commands to verify fix
7. **Lessons Learned** - What to remember
